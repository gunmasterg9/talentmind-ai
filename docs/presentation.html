The Problem

Why Traditional Sourcing Fails

01

Shallow Keyword Filtering

Legacy ATS systems match on exact words — missing a "Machine Learning Engineer" profile that writes "Deep Learning" and "Neural Networks" instead of "ML". Contextual synonymy is invisible to keyword scanners.

02

Ignoring Behavioral & Activity Signals

Resumes show what candidates claim, not what they do. Open-source contributions, recent certifications, hackathon wins, GitHub commits, and learning activity are powerful quality signals that go completely unused.

03

Rigid, Untunable Black-Box Rankings

Recruiters receive a binary yes/no list with no ability to adjust what matters most — for a startup, activity signals outweigh tenure; for an enterprise staff role, career growth trajectory is paramount. No existing tool exposes these controls.

04

No Explainability — No Trust

When AI gives a score of "82%", recruiters don't know why. Without human-readable reasoning, AI recommendations go ignored. Explainability is not optional — it's the difference between adoption and rejection.

The Solution

TalentMind AI: Multi-Signal Sourcing Engine

🧠

Deep Job Understanding via LangChain + NLP Parser

Structured extraction of required skills, preferred skills, soft skills, experience range, seniority level, and industry domain from raw free-text job descriptions. Supports multi-format JD input (paste, upload, or API).

LangChain Regex Parser Structured Extraction
🔍

Semantic Vector Search — Beyond Keywords

Candidate profiles are embedded using BAAI/bge-large-en-v1.5 Sentence Transformers and stored in ChromaDB. JD embeddings are compared via cosine similarity to surface semantically relevant candidates that would be invisible to keyword filters.

Sentence Transformers ChromaDB Cosine Similarity
📡

Activity & Behavioral Signal Integration

Beyond static profiles — GitHub commits, LeetCode rankings, open-source PRs, Kaggle medals, recent certifications, and learning hours are all normalized and weighted into the final score, rewarding actively growing candidates.

GitHub Activity Certification Recency Career Trajectory
💬

Explainable AI — Human-Readable Recruiter Summaries

Every ranked candidate receives a structured XAI summary: specific strengths matched, skill gaps identified, and a clear STRONG HIRE / CONSIDER / PASS recommendation. No black boxes, no guessing.

Core Algorithm

The Intelligent Ranking Formula

Final Score = 0.35 × Ssemantic + 0.20 × Sexperience + 0.15 × Sskills + 0.10 × Sindustry + 0.10 × Sactivity + 0.10 × Sgrowth
All weights are normalized (sum = 1.00) and recruiter-tunable via the dashboard UI
🔍 Semantic Match
35%
📅 Experience Match
20%
🛠️ Skills Overlap
15%
🏭 Industry Domain
10%
⚡ Activity Signals
10%
📈 Career Growth
10%

🔍 Semantic Score

Cosine similarity between 1024-dim vectors of JD text and candidate profile text. Captures conceptual alignment beyond surface keywords.

🛠️ Skills Score

Weighted Jaccard index: required skills carry 2× weight vs. preferred skills. Partial credit for subset matches.

⚡ Activity Score

Normalized composite of: GitHub commits, open-source PRs, LeetCode rank, Kaggle medals, certifications & learning hours in past 12 months.

📈 Growth Score

Career trajectory: company tier promotions (startup → unicorn → FAANG), title seniority progression evaluated over career history.
